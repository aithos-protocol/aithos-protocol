# 8 · Glossary

A canonical list of terms. Alphabetical.

---

**Action artifact.** A signed JSON object emitted by an agent when it acts under a mandate. Names the mandate, describes the action, carries the agent's signature (and optionally a counter-signature from the mandated sphere key). Defined in §5.4.

**Actor sphere.** The sphere (public / circle / self) whose key a mandate authorizes the agent to *speak as*. Always equals the sphere that signed the mandate. Defined in §4.2.2.

**Agent.** Any software client that fetches an ethos or acts under a mandate. Not a synonym for "AI"; a plain script can be an agent.

**AEAD.** Authenticated Encryption with Associated Data. Aithos uses XChaCha20-Poly1305-IETF. Defined in §3.4.

**Argon2id.** The passphrase key derivation function used to seal seeds and optionally zone keys at rest. Parameters: memlimit ≥ 64 MB, opslimit ≥ 3. §1.4.3.

**Author.** An implementation role: produces bundles and mandates. §0.4.2.

**Authorization header scheme.** `Authorization: Aithos <mandate-token>`. §6.3.4.

**Binding action.** An action whose scope appears in the mandate's `constraints.require_counter_sign`. Cannot be taken without a live counter-signature from the mandated sphere key. §5.4.3.

**Bundle.** A `.ethos` zip archive carrying an ethos edition. Chapter 3.

**Bundle ID.** `urn:aithos:<handle>:<edition_version>`. The globally unique identifier for an edition of an ethos. §0.6.4.

**Canonical form.** A JCS-canonicalized (RFC 8785) byte sequence of a JSON object with the signature field blanked. What gets signed. §5.1.

**Canonical URL.** The authoritative URL at which an edition is served. Optional; when present, part of `edition`. §2.4.

**Circle zone.** The middle disclosure zone. Encrypted. Readable by the subject and by anyone holding a corresponding key or a valid mandate with `ethos.read.circle`. §2.5 / chapter 3.

**Constraints.** An optional object on a mandate carrying non-normative hints that narrow the mandate further (domain restrictions, rate limits, counter-sign requirements). §4.2.5.

**Content hash.** `sha256:` followed by the hex SHA-256 of an action's payload. Present in action artifacts so the artifact commits to the content without including it. §5.4.2.

**Counter-signature.** A second signature on an action artifact, produced by the mandated sphere key. Required for binding actions. §5.4.3.

**DEK (Data Encryption Key).** A fresh 32-byte symmetric key used to encrypt a zone's plaintext. Wrapped per-recipient. §3.4.

**Delegate key.** An Ed25519 key pair generated by the subject for a specific agent or device, named in a write mandate's `grantee.pubkey`. Signs gamma entries (§10) on behalf of a sphere key without ever holding the sphere key itself. §4.5.4.

**DID.** Decentralized Identifier. An identifier resolvable to a DID document. Aithos uses method `aithos`. Chapter 1.

**DID document.** A JSON-LD object describing a DID's keys and services. For `did:aithos`, signed by the root key. §1.5.

**DID URL.** A DID plus optional path, query, fragment. Aithos uses fragments to name sphere keys (`#public`, `#circle`, `#self`, `#public-kex`, `#circle-kex`, `#self-kex`). §1.2.1.

**Ed25519.** The digital signature algorithm used throughout the protocol. RFC 8032. 32-byte public key, 64-byte signature. §5.2.

**Edition.** A specific published version of an ethos, immutable after signing. Linked to the previous edition by `edition.prev_hash`. §2.4, §3.3.3.

**Edition chain / per-edition spine.** The SHA-256 hash chain formed by `edition.prev_hash` on each consecutive bundle. §2.8.2, §3.3.3.

**Genesis edition.** An edition with `prev_hash: null`. Starts a new chain; optionally names its predecessor identity in `migrated_from`. §3.3.4.

**Height (edition).** The ordinal position of an edition in its chain. 1 for genesis, N for the Nth edition. §3.3.3.

**History spine.** The combination of per-section and per-edition hash chains that makes an ethos tamper-evident. §2.8.

**Ephemeral public key.** The X25519 public half of a one-shot key pair used to wrap a DEK for a recipient. §3.6.

**Ethos.** The signed, zone-partitioned artifact describing a subject. Either in document form (JSON, §2.2) or bundle form (zip, chapter 3).

**Free sections.** The design choice that zones contain an ordered array of user-titled sections rather than fixed schema slots. §2.7.

**Grantee.** The agent being granted by a mandate. Identified by `id` and optionally bound to a specific `pubkey`. §4.2.4.

**Handle.** A short, DNS-friendly identifier for a subject within a namespace. Not globally unique; the DID is. §2.3.

**HKDF.** HMAC-based Key Derivation Function. Used in key wrapping to derive a wrap key from an ECDH shared secret. §3.6.

**Holder.** Any party presenting a credential (a mandate) to access a non-public zone. §0.3.

**Id (mandate / action).** `mandate_<ULID>` or `action_<ULID>`. 26-character Crockford-base32 ULID. Time-orderable. §4.2.3, §5.4.2.

**Issuer.** The subject root DID on a mandate or revocation. §4.2.2.

**Issued-by-key.** The sphere DID URL that signed a mandate or revocation. MUST equal the sphere corresponding to `actor_sphere`. §4.4.

**JCS.** JSON Canonicalization Scheme, RFC 8785. §5.1.

**Key agreement.** X25519 ECDH. Used to wrap DEKs. §3.6.

**Keystore.** The local storage layout for keys and mandates. Informative; defined in §1.7.

**Manifest.** `manifest.json` inside a bundle. The single source of truth for what's in the bundle. §3.3.

**Mandate.** A signed capability token granting authority to read zones and take actions. Chapter 4.

**Multibase.** A self-describing encoding format. Aithos uses `z` (base58btc) for keys, consistent with `did:key`. §1.2.

**Multicodec.** A self-describing byte-prefix family. Ed25519 public keys use `0xed 0x01`; X25519 use `0xec 0x01`. §1.2.

**Nonce (mandate).** Random value ≥ 48 bits in a mandate, used to make the signed value unpredictable and to defeat replay of captured mandates. §4.2.2.

**Nonce (AEAD).** 24-byte random value for XChaCha20-Poly1305-IETF, unique per message. §3.4.

**Not-before / not-after.** The validity window of a mandate. §4.2.2.

**Public zone.** The disclosure zone readable by anyone without authentication. Not encrypted. Integrity protected by SHA-256 hash + signature. §2.5, §3.3.

**Reader.** An implementation role: consumes bundles. §0.4.2.

**Recipient.** A DID URL listed in a zone's `wraps` array, able to decrypt the zone. §3.5.

**Redaction revision.** A special revision whose body is a structured marker indicating that a past revision has been withdrawn. Logged, dated, signed. §2.5.4.3.

**Revision.** A single dated, signed entry in a section's append-only history log. Carries `prev_hash` linking to the previous revision. §2.5.4.

**Revocation.** A signed document withdrawing a mandate. Not retroactive. §4.6.

**Revocation list.** An aggregated, signed list of active revocations. §4.6.4.

**RFC 3339.** Timestamp format used throughout the protocol. UTC, `Z` suffix, second precision.

**Root DID.** `did:aithos:<multibase-ed25519>`. The subject's primary identifier. §1.2.

**Root key.** The Ed25519 key whose public half is embedded in the subject's root DID. Signs only the DID document.

**Scope.** A capability string from the controlled vocabulary in §4.3. One mandate may carry many scopes.

**Section.** An ordered element within a zone, carrying a title and a body. §2.5.1.

**Self zone.** The innermost disclosure zone. Readable only by the subject. §2.5.

**Signature.** An Ed25519 signature object with `alg`, `key`, and `value` fields. Value is base64url of the 64-byte signature. §5.2.

**Sphere.** One of the three zones (`public`, `circle`, `self`) as a cryptographic role. Each sphere has its own Ed25519 and X25519 key pair.

**Sphere key.** The Ed25519 key associated with a sphere. Signs operations within that sphere.

**Subject.** The person an ethos describes. Exactly one subject per ethos. §0.3.

**Superseded.** An edition that has been replaced by a later edition. Remains immutable and verifiable. §2.4.

**Tags (section).** Optional free-form tags on a section. Informative. §2.5.1.

**TTL.** The `not_after - not_before` interval of a mandate. Expressed colloquially as "7d", "24h", etc. in CLI.

**ULID.** [Universally Unique Lexicographically Sortable Identifier](https://github.com/ulid/spec). 26-character Crockford-base32 string. §4.2.3.

**URN (Aithos).** `urn:aithos:<handle>:<edition_version>`. The ethos `id`. §0.6.4.

**Verifier.** An implementation role: validates signatures. §0.4.2.

**Wrap.** An encrypted DEK bound to a single recipient, so that only that recipient can recover the DEK. §3.6.

**Write mandate.** A mandate carrying one of `ethos.write.public`, `ethos.write.circle`, `ethos.write.self`. Names a delegate key that is authorized to author gamma entries (§10) affecting the zone without holding the sphere key. §4.5.4.

**X25519.** Elliptic-curve Diffie-Hellman on Curve25519. Used for key wrapping. Distinct from Ed25519 (signature) but derivable from the same seed. §1.3.

**Zone.** One of the three disclosure partitions: `public`, `circle`, `self`. §2.5.

**Zone document.** The projection of an ethos document restricted to one zone. What gets signed when a zone signature is computed. §5.1.3.
